01 / verdict

这东西好不好用？答案是：能用，但要按“工具库”来用

CLI-Anything 的价值不在于替代 Agent，也不在于让每个 CLI 都立刻稳定可用。它更像一个本地 CLI 工具发现层：Agent 先通过 CLI-Hub 找到候选工具，再看说明、安装、跑最小样例，最后把真正可用的工具沉淀成固定工作流。

最适合它的场景，是“我不知道本地有没有合适 CLI，但我想让 Agent 自己找、自己试、自己留下可复用口令”。

3个样本跑通：openscreen、mermaid、3MF。
2+个样本暴露环境或打包问题，需要二次修。
5步主流程：search、info、install、smoke、handoff。
1条核心建议：先验工具，再接入生产流程。

这次最明确的结论是：CLI-Anything 适合作为“Agent 的本地工具雷达”，不适合把所有搜索结果都当成已经可用的正式能力。它需要一层验收纪律。

03 / workflow

CLI-Hub 的正确用法：不要跳过 smoke test

这次验证下来，比较稳的方式不是“搜到就用”，而是把 CLI-Hub 当作工具目录。Agent 应该先用关键词搜索，再查看工具说明，安装以后只跑最小样例，确认输出文件和 JSON 都正常，最后再写进交接口令。

Step 1搜索候选工具，比如 video、diagram、3d。
Step 2读取 info，确认参数、依赖和输出格式。
Step 3安装工具，不把安装成功等同于可用。
Step 4跑 smoke test，检查真实输出文件。
Step 5沉淀口令，交给其他线程复用。
pip install cli-anything-hub
cli-hub search video
cli-hub info openscreen
cli-hub install openscreen
cli-anything-openscreen --help

以后交代给其他 Agent 的最短口令可以是：先用 cli-hub search 找候选，再用 cli-hub info 看说明，安装后必须跑一个最小样例，只有能产生真实文件或结构化输出的 CLI 才能写进工作流。

样本一：openscreen 跑通，适合视频探测和轻量渲染

05 / sample

样本二：mermaid 最适合给文章和文档补流程图

mermaid 是本次最值得优先沉淀的工具。它产出的不是“看起来成功”的日志，而是能直接放进文章、报告、App 说明里的图形资产。对于多线程协作，流程图尤其适合承载交接。

{
  "output": "F:\\codex\\reports\\cli-anything-test-more\\workflow2.svg",
  "format": "svg",
  "method": "mermaid-renderer",
  "file_size": 13381,
  "action": "render"
}

这里有个实用经验：SVG 可以作为源文件保留，但给公众号、长文和跨平台发布时，最好另转成 PNG。它牺牲一点可编辑性，换来图片加载和样式兼容的稳定。

06 / sample

样本三：3MF 证明它也能处理垂直格式

3MF 不是大众工具，但它证明 CLI-Anything 的目录不只覆盖常见办公和网页场景，也能触到 3D、制造、模型检查这类垂直格式。对 Agent 来说，这类工具的价值是把“我看不懂的二进制/工程文件”转成可判断的结构化结果。

它的适用边界也清楚：只在有 3D 文件检查、模型验收、制造数据审查时启用。普通文章创作线程不需要常驻它。

07 / limits

失败样本更重要：它提醒我们不能盲信工具目录

这次有些工具并没有直接跑通。它们不代表 CLI-Anything 不好用，反而说明这个系统必须配合“验收分层”：发现是发现，可用是可用，生产可用又是另一层。

hacker-feeds-cli

遇到 ModuleNotFoundError: cli_anything.hacker_feeds_cli。暂时不能交给内容选题线程直接用。

obsidian-cli

需要本机 Obsidian 和 vault 环境。方向很有价值，但要先让 Windows/知识库线程把依赖补齐。

clibrowser

可能需要 Rust toolchain 或额外 binary。先作为候选，不进入默认浏览器自动化流程。

libreoffice

需要 LibreOffice 本体。适合文档转换，但要先确认安装路径和 headless 调用方式。

n8n

需要本地 n8n 服务或安装。更像自动化平台入口，不是单次任务里的轻 CLI。

共性问题

目录命中不等于环境就绪。交给其他线程前，必须附带最小命令和成功输出路径。

08 / handoff

给其他 Agent 的交接口令

以后你可以直接这样交代给其他线程。它们不需要重新理解整篇报告，只要照着这个验收模式跑。

用 CLI-Anything / CLI-Hub 找本地 CLI 工具：
1. 先运行 cli-hub search <关键词> 找候选；
2. 再运行 cli-hub info <工具名> 看依赖、参数、输出；
3. 安装后必须跑 smoke test；
4. 只有产生真实文件或结构化 JSON 的工具，才能写入正式工作流；
5. 当前优先可用工具：mermaid、openscreen、3mf；
6. hacker-feeds-cli、obsidian-cli、clibrowser、libreoffice、n8n 先当候选，补齐环境后再验。
